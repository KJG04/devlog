## Keep - 현재 만족하고 있는 부분

- 짧은 시간이었지만 필수 구현과 추가 구현을 모두 구현한 점이 좋았다.
- 짧은 시간에 많은 기능들을 구현하면서 재미있었다.
- 필수와 도전을 모두 구현한 점이 좋았다.
- 도전적인 기능을 구현할 수 있어서 좋은 경험이 되었다.

## Problem - 불편하게 느끼는 부분

- 아쉽게도 테스트 코드를 작성하지 못했고 또한 코드 컨벤션을 나만 지키지 못했기에 이 점이 많이 아쉽다. 그리고 마지막에 확인했는데 카드를 수정할 때 리스트도 수정할 수 있게 하려고 했는데 그 점을 넣지 못했다는 것을 확인했다.
- 테스트 코드를 작성해서 pr 시 커버리지가 %이하이면 pr을 취소하는 것도 넣고 싶었는데 못했다.
- 급하게 진행하다 보니 중간 중간 디테일을 놓치는 점이 아쉽고 테스트 코드를 작성 못한 것이 아쉬웠다.
- 시간이 부족해 테스트 코드를 작성하지 못한것이 아쉽다.

## Try - Problem에 대한 해결책, 당장 실행 가능한 것

- 시간을 잘 분배해서 테스트 코드까지 작성할 수 있도록 하고, 코드 작성 처음부터 코드 컨벤션을 잘 지켜서 만드는 것이 중요할 것 같다. 코드 작성 완료 후 여러번 코드를 확인해서 빠진 조건이 있는지 체크를 해야할 것 같다.
- 프로젝트 시작전에 CI CD를 구축해놓고 테스트 코드도 바로바로 작성해야겠다.
- 시간 계획을 잘 세워서 빠지는 부분 없이 작성할 수 있도록 해야되겠다.
- 테스트 코드는 기능을 구현 할때 작성하여 추후 생각할 필요 없도록 하겠다.
